# 🏢 Step 3: Instalasi Secondary Domain Controller (DC2)

## Tujuan

Menginstal DC2 sebagai Secondary (Replica) Domain Controller untuk domain `corp.local`, menyediakan redundansi dan simulasi lingkungan enterprise yang realistis.

Server ini menggunakan **dua network adapter** seperti DC1:
- **Adapter PUBLIC** — IP public untuk akses RDP dari luar
- **Adapter INTERNAL** — IP private untuk AD DS, DNS, dan semua traffic domain

---

## 3.1 Membuat VM / Menyiapkan Server

| Parameter | Nilai |
|-----------|-------|
| Name | DC2 |
| OS | Windows Server 2019/2022 |
| RAM | 2 GB (atau 1.5 GB jika RAM terbatas) |
| CPU | 2 vCPU |
| Disk | 40 GB |
| Adapter 1 | PUBLIC — IP dari ISP/cloud provider (untuk RDP) |
| Adapter 2 | INTERNAL — Host-Only / private network (untuk AD) |

## 3.3 Identifikasi dan Rename Adapter

```powershell
Get-NetAdapter | Select-Object Name, InterfaceIndex, Status, MacAddress

Rename-NetAdapter -Name "Ethernet" -NewName "PUBLIC"
Rename-NetAdapter -Name "Ethernet 2" -NewName "INTERNAL"

Get-NetAdapter | Select-Object Name, Status
```

---

## 3.4 Konfigurasi IP Static — Dual Adapter

### Adapter PUBLIC

```powershell
New-NetIPAddress -InterfaceAlias "PUBLIC" `
    -IPAddress <IP_PUBLIC_DC2> `
    -PrefixLength 24 `
    -DefaultGateway <GATEWAY_PUBLIC>
```

**Hanya adapter PUBLIC yang boleh punya default gateway.**

### Adapter INTERNAL

```powershell
New-NetIPAddress -InterfaceAlias "INTERNAL" `
    -IPAddress 192.168.56.11 `
    -PrefixLength 24
# JANGAN set gateway
```

### DNS Client — Menunjuk ke DC1 (Bukan Diri Sendiri!)

**Ini perbedaan utama dengan DC1.** Sebelum join domain, DNS harus menunjuk ke DC1 agar DC2 bisa menemukan domain `corp.local`:

```powershell
Set-DnsClientServerAddress -InterfaceAlias "INTERNAL" -ServerAddresses 192.168.56.10
Set-DnsClientServerAddress -InterfaceAlias "PUBLIC" -ServerAddresses 192.168.56.10
```

---

## 3.5 Matikan DNS Registration di Adapter PUBLIC

```powershell
Set-DnsClient -InterfaceAlias "PUBLIC" -RegisterThisConnectionsAddress $false
```

---

## 3.6 Set Binding Order

```powershell
Set-NetIPInterface -InterfaceAlias "INTERNAL" -InterfaceMetric 10
Set-NetIPInterface -InterfaceAlias "PUBLIC" -InterfaceMetric 50
```

---

## 3.7 Ubah Hostname

```powershell
Rename-Computer -NewName "DC2" -Restart
```

---

## 3.8 Verifikasi Konektivitas ke DC1

**Jangan lanjut sebelum semua ini berhasil:**

```powershell
ping 192.168.56.10
nslookup corp.local
nslookup dc1.corp.local
Test-NetConnection 192.168.56.10 -Port 389
Test-NetConnection 192.168.56.10 -Port 88
```

---

## 3.9 Instal AD DS dan Promosikan sebagai Replica DC

```powershell
Install-WindowsFeature -Name AD-Domain-Services -IncludeManagementTools

Install-ADDSDomainController `
    -DomainName "corp.local" `
    -InstallDns:$true `
    -Credential (Get-Credential) `
    -SafeModeAdministratorPassword (ConvertTo-SecureString "Lab@dmin2024!" -AsPlainText -Force) `
    -Force:$true
```

Credential: `CORP\Administrator` + password domain.

**Catatan:** Perintahnya `Install-ADDSDomainController`, **BUKAN** `Install-ADDSForest`. DC1 membuat forest, DC2 hanya join sebagai replica.

---

## 3.10 Konfigurasi DNS Server — Hanya Listen di IP Internal

Setelah restart, login sebagai `CORP\Administrator`:

```powershell
dnscmd localhost /ResetListenAddresses 192.168.56.11
Restart-Service DNS
```

Verifikasi:

```powershell
dnscmd /Info /ListenAddresses
```

**Output yang diharapkan:** Hanya `192.168.56.11`.

Jika dnscmd gagal, pakai registry:

```powershell
Set-ItemProperty -Path "HKLM:\SYSTEM\CurrentControlSet\Services\DNS\Parameters" `
    -Name "ListenAddresses" -Value @("192.168.56.11")
Restart-Service DNS
```

---

## 3.11 Update DNS Client di DC2 (Setelah Promosi)

Sekarang DC2 punya DNS sendiri, update agar fallback ke diri sendiri:

```powershell
Set-DnsClientServerAddress -InterfaceAlias "INTERNAL" -ServerAddresses 192.168.56.10, 192.168.56.11
Set-DnsClientServerAddress -InterfaceAlias "PUBLIC" -ServerAddresses 192.168.56.10, 192.168.56.11
```

---

## 3.12 Update DNS Client di DC1 (Crossover Fallback)

Kembali ke **DC1**, update agar fallback ke DC2:

```powershell
Set-DnsClientServerAddress -InterfaceAlias "INTERNAL" -ServerAddresses 192.168.56.10, 192.168.56.11
Set-DnsClientServerAddress -InterfaceAlias "PUBLIC" -ServerAddresses 192.168.56.10, 192.168.56.11
```

---

## 3.13 Bersihkan DNS Record yang Salah

Cek di **DC1** — DNS Manager (`dnsmgmt.msc`) → Forward Lookup Zones → corp.local:
- Cari A record `dc2` → jika menunjuk ke IP public, **hapus**
- Sisakan hanya record yang menunjuk ke `192.168.56.11`

Force re-register di **DC2**:

```powershell
ipconfig /flushdns
ipconfig /registerdns
```

Verifikasi dari DC1:

```powershell
nslookup dc2.corp.local
# Harus mengembalikan: 192.168.56.11
```

**PENTING:** Jika `nslookup dc2.corp.local` dari DC1 mengembalikan IP public atau gagal, replikasi AD **akan error** (`DNS lookup failure`). Fix ini dulu sebelum lanjut.

## 3.14 Verifikasi Replikasi

```powershell
Get-ADDomainController -Filter * | Select-Object Name, IPv4Address, IsGlobalCatalog

repadmin /replsummary
repadmin /showrepl
repadmin /syncall /AdeP
```

Output `repadmin /replsummary` harus menunjukkan **0 failures**.

## Ringkasan Perbandingan DC1 vs DC2

| Konfigurasi | DC1 | DC2 |
|-------------|-----|-----|
| IP Internal | 192.168.56.10 | 192.168.56.11 |
| DNS Client (sebelum promosi) | 192.168.56.10 (diri sendiri) | 192.168.56.10 (DC1) |
| DNS Client (setelah promosi) | 192.168.56.10, 192.168.56.11 | 192.168.56.10, 192.168.56.11 |
| Perintah promosi | `Install-ADDSForest` | `Install-ADDSDomainController` |
| DNS Listen Address | 192.168.56.10 | 192.168.56.11 |
| Reverse lookup zone | Buat manual | Otomatis replikasi dari DC1 |

## Mengapa Perlu Secondary DC?

1. **Redundansi** — jika DC1 mati, autentikasi tetap berjalan
2. **Load balancing** — request autentikasi didistribusi
3. **Realisme lab** — mempraktikkan DCSync attack
4. **Attack surface tambahan** — lateral movement dari/ke DC kedua

---

## Troubleshooting

### Replikasi error "DNS lookup failure"

Ini error paling umum saat setup DC2 dual-NIC. Penyebabnya: A record `dc2` di DNS menunjuk ke IP public.

Fix:
1. Di DC1: `nslookup dc2.corp.local` → cek apakah resolve ke IP public
2. Jika ya, hapus A record yang salah di DNS Manager
3. Buat ulang: `Add-DnsServerResourceRecordA -ZoneName "corp.local" -Name "dc2" -IPv4Address 192.168.56.11`
4. Di DC2: `ipconfig /flushdns && ipconfig /registerdns`
5. Test lagi: `repadmin /replsummary`

### Gagal join domain / promosi gagal

- Pastikan DNS menunjuk ke `192.168.56.10` (DC1)
- Cek `nslookup corp.local` — harus resolve
- Cek `Test-NetConnection 192.168.56.10 -Port 389`

### DNS zone tidak muncul di DC2

- Tunggu beberapa menit untuk replikasi
- Force: `repadmin /syncall /AdeP`
- Restart DNS: `Restart-Service DNS`
